**2b — Create scripts/generate-icons.ts:**
```typescript
import sharp from 'sharp';
import path from 'path';
import fs from 'fs';

const SIZES = [192, 512];
const BG_COLOR = { r: 255, g: 249, b: 245, alpha: 1 }; // #FFF9F5
const OUTPUT_DIR = path.join(process.cwd(), 'public', 'icons');

async function generateIcon(size: number): Promise<void> {
  fs.mkdirSync(OUTPUT_DIR, { recursive: true });

  const fontSize = Math.round(size * 0.5);
  const svg = `<svg width="${size}" height="${size}" xmlns="http://www.w3.org/2000/svg">
    <text x="50%" y="55%" font-size="${fontSize}" text-anchor="middle" dominant-baseline="middle">🎁</text>
  </svg>`;

  await sharp({
    create: { width: size, height: size, channels: 4, background: BG_COLOR },
  })
    .composite([{ input: Buffer.from(svg), top: 0, left: 0 }])
    .png()
    .toFile(path.join(OUTPUT_DIR, `icon-${size}.png`));

  console.log(`Generated icon-${size}.png`);
}

Promise.all(SIZES.map(generateIcon)).catch((err) => {
  console.error('Icon generation failed:', err);
  process.exit(1);
});
```

Add a `generate:icons` script to package.json:
```json
"generate:icons": "npx ts-node --esm scripts/generate-icons.ts"
```

If `--esm` causes issues with the current ts-node version, try:

Or use tsx if available: `"generate:icons": "npx tsx scripts/generate-icons.ts"`

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Service Worker ↔ Network | SW intercepts all network requests; must not cache sensitive auth tokens in precache |
| Service Worker ↔ Client Page | Page sends postMessage to SW; SW acts on message type only |
| /manifest.webmanifest | Public endpoint; no sensitive data should appear in manifest fields |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-02-01 | Information Disclosure | Service worker precache | mitigate | `@ducanh2912/next-pwa` caches static JS/CSS only; Firestore tokens are in memory (never precached). Configure `runtimeCaching` to exclude `/api/` routes from cache. Verify `workboxOptions` does not cache auth cookies. |
| T-05-02-02 | Tampering | public/sw.js served stale | mitigate | Next.js sets `Cache-Control: no-store` on sw.js by default. Verify in response headers after `npm start`. If absent, add `headers()` rule in next.config.ts for `/sw.js`. |
| T-05-02-03 | Elevation of Privilege | Offline cache serving stale auth state | accept | Cached pages show stale UI data only; all auth actions (mark purchased, add note) require network requests. Firestore security rules remain the true auth gate. |
| T-05-02-04 | Denial of Service | Malformed manifest causing install failure | accept | Manifest is a static build artifact; no user-controlled input; worst case is install prompt not showing (non-security). |
| T-05-02-05 | Information Disclosure | public/icons exposed to public | accept | Icons are intentionally public assets; no secrets; background color and emoji are not sensitive. |
</threat_model>
