## Interaction Contracts

### Purchased State Toggle (D-10)

- Control: checkbox input styled as a toggle button, or a button with checkmark icon
- Unchecked appearance: border `#E5D5CC`, background white, label "Markera som köpt"
- Checked appearance: background `#F97316`, white checkmark icon, label "Markerad som köpt"
- When purchased by another viewer: toggle is disabled (greyed, `opacity-50`, `cursor-not-allowed`); label shows "Köpt av [name]"
- Purchased card treatment: item title gets `line-through` + `text-[#6B7280]`; card background remains `#FFF0E8` (no color change, just muted text)
- Undo: if current viewer is the purchaser, toggle remains clickable to unmark

### Viewer Note Field (D-13)

- Collapsed state (no note): shows a text button "Lämna en anteckning" at 14px/400, muted color `#6B7280`
- Expanded state: `<textarea>` with `rows={3}`, same border/background pattern as existing form inputs (`border-[#E5D5CC] rounded-md px-3 py-2 bg-white`); auto-saves on blur (no separate save button)
- Note exists (own note): shows inline truncated text at 14px/400 in `#171717`, with "Redigera" link at 14px accent color; click expands to textarea
- Transition: CSS `transition-all duration-150` on height for expand/collapse — no JS animation library

### Other Viewers' Notes (D-14)

- Rendered below own note field when other notes exist
- Each entry: `"[username]: [note text]"` at 14px/400, color `#6B7280`, read-only (no interaction)
- Max display: show all notes (no truncation per note; long notes wrap)

### Viewer Dashboard Grid (D-20)

- Layout: CSS grid, 1 column on 375px, 2 columns on 640px+
- Each `WishlistDashboardCard`: `bg-[#FFF0E8] border border-[#E5D5CC] rounded-2xl shadow-sm p-4`
- Shows: child name as heading (20px/600), total items count as label (14px/400 muted), purchased count as `"X av Y köpta"` at 14px/400 muted, placeholder thumbnail `div` if no image
- Click target: entire card is a link (`<a>`) to `/viewer/[wishlistId]`
- No purchased items visible (count only, no spoilers)

### Invite Page (D-03, Specifics)

- Welcoming tone: heading "Du har bjudits in till [barnets namn]s önskelista" at 28px/600
- Below heading: description at 16px/400 explaining they need an account to coordinate
- Two inline options: "Logga in" (outline button) and "Skapa konto" (accent button), same row on mobile
- Already logged in + already viewer: auto-redirect, no UI rendered
- Already logged in + not yet viewer: show "Gå med" confirmation button (accent), no auth form

### Activity Log (D-16, D-17)

- Route: `/viewer/[wishlistId]/activity`, linked from viewer page header
- Each entry: row with viewer username (14px/600), action text (14px/400), item title (14px/400 italic), timestamp (12px/400 muted)
- Ordered newest-first
- Pagination: show 50 entries per page; "Visa fler" text button at bottom if more exist (no infinite scroll)
- Empty state: "Inga händelser ännu" centered at 16px/400 muted

### Settings Page Share Link Panel (D-01)

- Section heading: "Delningslänk" at 20px/600
- Current link: monospace-ish text input (read-only, selectable) + "Kopiera länk" accent button beside it
- Copy success feedback: button label changes to "Kopierat!" for 2 seconds, then reverts
- Viewer list: below the link, "Betraktare (X)" as subheading at 16px/600, then list of viewer display names at 14px/400
- "Generera ny länk" button: outline style (border `#E5D5CC`, text `#171717`), NOT accent — destructive pattern
- Regenerate confirmation: inline confirmation row ("Gamla länken slutar fungera. Fortsätt?" + "Ja, generera" (destructive red text link) + "Avbryt")

## Accessibility Contracts

- All interactive controls: minimum touch target 44px height (established Phase 3, maintained)
- "Mark as purchased" toggle: `aria-label="Markera [titel] som köpt"` / `aria-label="Avmarkera [titel]"` dynamically
- Purchased card state: `aria-label` on card includes "(köpt av [namn])" for screen reader context
- Copy button: `aria-live="polite"` region announces "Kopierat!" on success
- Invite page auth forms: same `aria-required` and `role="alert"` patterns as Phase 2 auth forms
- Activity log timestamps: use `<time datetime="{ISO}">` element
- Settings regenerate confirmation: inline `role="alert"` so screen readers announce the confirmation step

## State Inventory

| State | Trigger | Visual Treatment |
|-------|---------|-----------------|
| Loading (viewer wishlist) | Initial data fetch | `LoadingSkeleton` (reused from Phase 3) |
| Loading (dashboard) | Initial data fetch | Grid of 2 skeleton cards at `#FFF0E8` |
| Empty (dashboard) | No wishlists found | EmptyState pattern, centered, no CTA button |
| Error (viewer wishlist) | Firestore read fails | Red text error at top of page, retry text link |
| Purchased (card) | `purchasedBy` field set | Strikethrough title, muted text, toggle shows purchaser name |
| Note expanded | User clicks "Lämna en anteckning" | Textarea appears below existing content, 150ms height transition |
| Note saved | Auto-save on blur | No visual change (silent save); spinner during in-flight |
| Note save error | Firestore write fails | Inline red text below textarea: "Anteckningen kunde inte sparas. Försök igen." |
| Invalid invite | Token `active: false` or not found | Full-page error state with heading + body copy from copywriting contract |
| Copy success | User clicks "Kopiera länk" | Button label changes to "Kopierat!" for 2s |
| Regenerate confirm | User clicks "Generera ny länk" | Inline confirmation row appears below button |
