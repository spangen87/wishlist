## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding, gap between label and input (mb-1) |
| sm | 8px | Compact element spacing |
| md | 16px | Default element spacing, form gap (gap-4), page padding (p-4) |
| lg | 24px | Section padding (p-6), vertical form section margins |
| xl | 32px | Layout gaps, heading bottom margin (mb-6 = 24px, mb-8 = 32px) |
| 2xl | 48px | Empty state vertical padding (py-16 = 64px, nearest 2xl reference) |
| 3xl | 64px | Page-level spacing |

Exceptions:
- Touch targets: all interactive elements (buttons, links used as buttons) must have `min-h-[44px]` — this is established in login/dashboard and must be maintained on all new interactive elements in this phase.
- Wizard card: uses `max-w-sm` container (384px), centered, `p-4` outer page padding — matches login/register pattern.

Source: `src/app/login/page.tsx`, `src/app/dashboard/page.tsx`, `src/components/viewer/ShareLinkPanel.tsx`.

## Typography

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 16px (text-base) | 400 regular | 1.5 |
| Label | 14px (text-sm) | 500 medium (font-medium) | 1.5 |
| Heading (page/step) | 24px (text-2xl) | 700 bold (font-bold) | 1.2 |
| Display (dashboard section heading) | 28px (text-[28px]) | 600 semibold (font-semibold) | 1.2 |

Notes:
- Wizard step heading uses 24px bold, centered — matches existing `<h1 className="text-2xl font-bold mb-6 text-center">` in login/register.
- Section sub-headings (e.g. within ShareLinkPanel) use 20px semibold (text-xl font-semibold) — matches existing panel h2 in ShareLinkPanel.
- Error/muted text uses 14px regular weight — matches existing `text-sm` error paragraphs.
- The display role (28px) is reserved for empty state headings on the dashboard — matches existing `text-[28px] font-semibold` in dashboard empty state.

Source: `src/app/login/page.tsx` (h1 text-2xl font-bold), `src/app/dashboard/page.tsx` (h2 text-[28px] font-semibold, h1 text-xl font-semibold), `src/components/viewer/ShareLinkPanel.tsx` (h2 text-xl font-semibold).

## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | #FFF9F5 (--color-bg) | Page background, min-h-screen base |
| Secondary (30%) | #FFF0E8 (--color-card) | Cards, panels, form containers, ShareLinkPanel background |
| Accent (10%) | #F9A87A (--color-accent), hover #F49060 (--color-accent-hover) | See reserved list below |
| Destructive | #DC2626 (--color-destructive) | Destructive action buttons and error messages only |
| Border | #E5D5CC (--color-border) | Input borders, card borders, divider lines |
| Text primary | #171717 (--color-text) | All body and heading text |
| Text muted | #6B7280 (--color-muted) | Helper text, secondary labels, loading messages, viewer counts |

Accent reserved for:
- Primary CTA submit buttons on all forms (wizard steps, /add-child)
- The "Kopiera länk" button inside ShareLinkPanel (Step 3)
- Inline text links (e.g. "Registrera dig som betraktare" on login page, "Logga in" on register page)

Accent is NOT used for:
- Secondary/ghost buttons (use border-[#E5D5CC] outline style — matches "Generera ny länk" pattern)
- Destructive confirmations (use --color-destructive)
- The "Lägg till barn" button in the dashboard (see Copywriting section for treatment)

Source: `src/app/globals.css` (all tokens), `src/app/login/page.tsx` (accent on primary CTA), `src/components/viewer/ShareLinkPanel.tsx` (accent on copy button, ghost style on secondary button).

## Component Contracts

### Wizard (/onboarding)

**Container:** `<main className="flex min-h-screen items-center justify-center p-4 bg-[#FFF9F5]"><div className="w-full max-w-sm">`
- Matches login/register pattern exactly. Source: `src/app/login/page.tsx`.

**Progress indicator:** Numbered step dots (3 dots, current step filled with accent color, remaining dots unfilled with border color).
- Dot size: 8px (w-2 h-2), active dot: bg-[#F9A87A], inactive dot: bg-[#E5D5CC]
- Positioned below heading, above form content, centered
- Accessible: `aria-label="Steg {N} av 3"` on the container, current step dot has `aria-current="step"`
- This is Claude's discretion per CONTEXT.md D-discretion item 1.

**Step transition:** Static (no animation). Controlled by local state, step component swaps in place. No CSS transitions between steps — keeps implementation simple and avoids hydration issues.
- This is Claude's discretion per CONTEXT.md D-discretion item 2.

**Wizard state persistence:** React `useState` at the `/onboarding` page level. After Step 1 creates child account, wishlistId is stored in state and passed to Step 2 and Step 3.

**Step 1 — Create barnkonto:**
- Heading: "Skapa barnkonto" (text-2xl font-bold mb-6 text-center)
- Fields (in order): Visningsnamn, Användarnamn, Lösenord, Ålder
- Field layout: same as login — `<label className="block text-sm font-medium mb-1">` + `<input className="w-full border border-[#E5D5CC] rounded px-3 py-2">`
- Age field: `type="number"` input, `min="1"` `max="18"`, label "Ålder"
- Primary CTA: "Skapa konto" — accent button, `min-h-[44px]`, loading state "Skapar…"
- Error display: `{error && <p role="alert" className="text-[#DC2626] text-sm">{error}</p>}` — positioned above submit button

**Step 2 — Name the wishlist:**
- Heading: "Namnge önskelistan" (text-2xl font-bold mb-6 text-center)
- Single field: "Namn på önskelistan" — text input, placeholder "t.ex. Elsas önskelista"
- Primary CTA: "Spara och fortsätt" — accent button, `min-h-[44px]`, loading state "Sparar…"
- Error display: same pattern as Step 1

**Step 3 — Share link:**
- Heading: "Dela önskelistan" (text-2xl font-bold mb-6 text-center)
- Content: `<ShareLinkPanel wishlistId={wishlistId} viewers={[]} />` — used as-is, no modification
- Below panel: "Gå till önskelistan" — accent button, `min-h-[44px]`, navigates to `/viewer/{wishlistId}`
- No error state needed at this step (ShareLinkPanel handles its own errors internally)

### Add Child Page (/add-child)

**Container:** Same `flex min-h-screen items-center justify-center p-4` pattern.
- Heading: "Lägg till barn" (text-2xl font-bold mb-6 text-center)
- Fields: identical to Wizard Step 1 — same shared form component (ChildAccountForm)
- Primary CTA: "Skapa konto" — same accent button
- Below form: back-link "Tillbaka till instrumentpanelen" — text-sm text-[#6B7280] underline, centered, margin-top 16px

### Dashboard — "Lägg till barn" button

**Placement:** Below the wishlists grid (or below empty state), above nothing — pinned at the bottom of the content flow, not floating.
- Style: ghost/outline button — `border border-[#E5D5CC] rounded-xl px-4 py-2 text-sm font-medium text-[#171717] hover:bg-[#FFF0E8] min-h-[44px] transition-colors`
- This is Claude's discretion per CONTEXT.md D-discretion item 4.
- Label: "Lägg till barn"
- Width: `w-full` on mobile, `w-auto` on sm breakpoint — left-aligned at sm+
