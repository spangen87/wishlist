<objective>
Block stored XSS via javascript:/data: URIs in productUrl and imageUrl fields. Two attack surfaces exist: the parent-facing add-item API route and the child's direct Firestore write path (addWishItem/updateWishItem in wishlist.ts). Client-side components add a defence-in-depth render guard for items already stored.

Purpose: Prevent any user from storing a javascript: URI that would execute in other viewers' browsers when they click the product link (SEC-02).
Output: URL scheme validation in add-item route, addWishItem, updateWishItem; isSafeUrl render guard in both card components; new test file covering the validated route.
</objective>

const SAFE_URL_PREFIXES = ['https://', 'http://'];
if (productUrl?.trim()) {
  const url = productUrl.trim();
  if (!SAFE_URL_PREFIXES.some(p => url.startsWith(p))) {
    return NextResponse.json(

      { status: 400 }
    );

  itemData.productUrl = url;
